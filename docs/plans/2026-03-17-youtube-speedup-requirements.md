# YouTube 轉錄加速 - 需求文件

**日期**: 2026-03-17
**版本**: 1.0
**狀態**: 已核准

---

## 1. 問題陳述

### 1.1 背景

目前 YouTube 影片轉錄功能速度較慢，對於 30-60 分鐘的影片，完整處理需要 5-10 分鐘。這影響使用者體驗，特別是在需要快速取得影片內容的場景。

### 1.2 現有流程瓶頸

```
YouTube URL → 下載音訊 (2-3 分鐘) → Whisper 轉錄 (3-7 分鐘) → 格式化輸出 (秒級)
```

**瓶頸分析**:
- **音訊下載**: 長影片音訊檔案較大
- **Whisper 轉錄**: CPU 環境下運算密集

### 1.3 目標

大幅減少 YouTube 影片轉錄時間，特別是對於已有字幕的影片。

## 2. 功能需求

### 2.1 FR-01: 字幕優先策略

**需求**: 系統應優先檢查並使用 YouTube 既有的字幕

**驗收標準**:
- [ ] 自動檢測影片是否有手動或自動字幕
- [ ] 有字幕時，直接下載並轉換為文字
- [ ] 無字幕時，降級到 Whisper 轉錄

### 2.2 FR-02: 字幕語言優先順序

**需求**: 系統應依語言優先順序選擇字幕

**語言優先順序**:
1. 手動字幕 > 自動字幕
2. 語言：`zh-Hant` > `zh-Hans` > `zh-TW` > `zh-CN` > `en`

**驗收標準**:
- [ ] 優先選擇手動上傳的字幕
- [ ] 依語言順序嘗試匹配
- [ ] 返回實際使用的語言資訊

### 2.3 FR-03: 慢速路徑優化

**需求**: 當無字幕需要 Whisper 轉錄時，應優化處理速度

**優化措施**:
- [ ] 支援下載低品質音訊 (64kbps)
- [ ] 支援多線程轉錄
- [ ] 維持 VAD 過濾功能

### 2.4 FR-04: API 參數控制

**需求**: 提供 API 參數讓使用者控制行為

**新增參數**:

| 參數 | 類型 | 預設值 | 說明 |
|------|------|--------|------|
| `prefer_subtitles` | bool | `true` | 優先使用 YouTube 字幕 |
| `model_size` | str | `base` | Whisper 模型大小 |
| `fast_mode` | bool | `false` | 啟用快速模式 |

**驗收標準**:
- [ ] `prefer_subtitles=true` 時優先檢查字幕
- [ ] `prefer_subtitles=false` 時強制使用 Whisper
- [ ] `fast_mode=true` 時啟用所有優化

### 2.5 FR-05: 回應格式增強

**需求**: 回應應包含處理來源資訊

**新增欄位**:

```json
{
  "metadata": {
    "source": "youtube_subtitles" | "whisper",
    "language": "zh-Hant",
    "is_auto_generated": false,
    "processing_time_ms": 2500
  }
}
```

**驗收標準**:
- [ ] `source` 欄位正確標識來源
- [ ] `language` 返回實際使用的語言
- [ ] `is_auto_generated` 標識字幕類型
- [ ] `processing_time_ms` 記錄處理時間

### 3.3 NFR-03: 向後相容

- 現有 API 參數應繼續有效
- 預設行為應為字幕優先

## 4. 使用案例

### UC-01: 快速轉錄有字幕影片

**前置條件**: 影片有 YouTube 自動字幕

**流程**:
1. 使用者呼叫 `/api/v1/convert/youtube?url=...`
2. 系統檢測到有 `zh-Hant` 字幕
3. 系統下載字幕並轉換
4. 返回轉錄結果

**預期結果**: 2-5 秒內完成

### UC-02: 轉錄無字幕影片

**前置條件**: 影片無任何字幕

**流程**:
1. 使用者呼叫 `/api/v1/convert/youtube?url=...`
2. 系統檢測無字幕
3. 系統下載音訊並用 Whisper 轉錄
4. 返回轉錄結果

**預期結果**: 3-5 分鐘內完成

### UC-03: 強制使用 Whisper

**前置條件**: 影片有字幕，但使用者想用 Whisper

**流程**:
1. 使用者呼叫 `/api/v1/convert/youtube?url=...&prefer_subtitles=false`
2. 系統跳過字幕檢查
3. 直接使用 Whisper 轉錄

**預期結果**: 使用 Whisper 轉錄

## 5. 限制與假設

### 5.1 限制

- CPU 環境（無 GPU 加速）
- 記憶體限制 2GB
- Docker 容器環境

### 5.2 假設

- yt-dlp 可正常存取 YouTube
- YouTube 字幕品質可接受
- 使用者可接受自動字幕的準確度
